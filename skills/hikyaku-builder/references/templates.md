# BUILDフェーズ テンプレート集

## plan.md（実装計画）

issue.md のスコープ定義・受け入れ基準をソースオブトゥルースとし、plan.md では実装固有の情報のみ記載する。

```markdown
# Build {NN}: {ビルド名}

## 依存パッケージ

| パッケージ | 用途 | 備考 |
|-----------|------|------|
| `package-name` | 用途の説明 | 新規追加 / バージョンアップ等 |

（新規追加・変更がなければ「なし」）

## クラス設計

（メソッドシグネチャを含む。architecture/interfaces.md で定義済みのものは差分のみ記載）

## 非機能要件

- **セキュリティ**: （認証・認可・入力バリデーション・SQLインジェクション対策等）
- **パフォーマンス**: （N+1防止・ページネーション・キャッシュ等）
- **エラーハンドリング**: （エラー種別・ユーザーへの表示方針等）

（該当しない項目は省略）

## BP見積もり

見積結果が6BP以上となった場合、`/hikyaku-build-manager {DOC_ROOT}`を呼び出してビルド分割を検討すること

- architectフェーズ段階: 〇BP 
- builderフェーズ段階: 〇BP

## 実装ステップ

- [ ] Step 1: ...
- [ ] Step 2: ...
- [ ] Step 3: ...
- [ ] Step N: ローカル検証（lint / typecheck / test / build）
```

## questions.md（ビルド固有の質問）

実装計画の作成中に判断できない点があれば、このファイルを作成してユーザーに質問する。

```markdown
# Build {NN}: {ビルド名} 質問

## Q1: （質問内容）
**回答**:

## Q2: （質問内容）
**回答**:
```

## handoff.md（申し送り）

後続ビルドのセッションがコンテキストを復元するためのドキュメント。設計ドキュメント（architecture/）に反映済みの情報は重複して書かない。

```markdown
# Build {NN}: {ビルド名} 申し送り

## 公開インターフェース

後続ビルドが使うインターフェースのうち、architecture/interfaces.md に反映済みのものは省略し、差分・補足のみ記載する。

- `ClassName.method(args): ReturnType` — 用途・補足
- `functionName(args): ReturnType` — 用途・補足

（interfaces.md に全て反映済みであれば「interfaces.md 参照」）

## 技術的判断の記録（ADR）

### ADR-1: （判断タイトル）
- **状況**: （何を決める必要があったか）
- **決定**: （何を選んだか）
- **理由**: （なぜそう判断したか）

（判断がなければ「なし」）

## 環境変更

- （新しい環境変数、DBマイグレーション、設定ファイルの追加等）

（変更がなければ「なし」）

## 既知の制約・注意点

- （後続ビルドが知るべき前提、制約、意図的に対応しなかった事項等）

（なければ「なし」）
```
