# BUILDフェーズ テンプレート集

## plan.md（実装計画）

```markdown
# Build {NN}: {タスク名}

## 概要

（このタスクで何を実現するかを1-2文で）

## やること

- （スコープ内の作業を箇条書き）

## やらないこと

- （明示的にスコープ外とするもの）

## 依存パッケージ

| パッケージ | 用途 | 備考 |
|-----------|------|------|
| `package-name` | 用途の説明 | 新規追加 / バージョンアップ等 |

（新規追加・変更がなければ「なし」）

## クラス図

（メソッドシグネチャを含む。architecture/interfaces.md で定義済みのものは差分のみ記載）

## 非機能要件

- **セキュリティ**: （認証・認可・入力バリデーション・SQLインジェクション対策等）
- **パフォーマンス**: （N+1防止・ページネーション・キャッシュ等）
- **エラーハンドリング**: （エラー種別・ユーザーへの表示方針等）

（該当しない項目は省略）

## 受け入れ基準

- [ ] （完了条件1）
- [ ] （完了条件2）
- [ ] （完了条件3）
- [ ] lint / typecheck / test / build が全てパスする

実装ステップ

- [ ] Step 1: ...
- [ ] Step 2: ...
- [ ] Step 3: ...
- [ ] Step N: ローカル検証（lint / typecheck / test / build）
```

## test-spec.md（テストシナリオ）

Agent が plan.md・issue.md・architecture/ を元に生成する。ユーザーが自然言語で「何をテストするか」を確認できる形式。

```markdown
# Build {NN}: {タスク名} テストシナリオ

## {テスト対象クラス/モジュール名}

### {メソッド名}: {シナリオ名}
- Given: （前提条件）
- When: （操作）
- Then: （期待結果）

### {メソッド名}: {シナリオ名}
- Given: （前提条件）
- When: （操作）
- Then: （期待結果）

## {テスト対象クラス/モジュール名}

### {メソッド名}: {シナリオ名}
- Given: （前提条件）
- When: （操作）
- Then: （期待結果）
```

**記載ガイドライン:**
- 正常系・異常系・境界値を網羅する
- Given/When/Then は具体的な値を含める（例: `Given: メールアドレス "user@example.com" のユーザーが登録済み`）
- 1シナリオ = 1つの検証観点に絞る

```markdown
### ADR-1: （判断タイトル）
- **状況**: （何を決める必要があったか）
- **決定**: （何を選んだか）
- **理由**: （なぜそう判断したか）

（判断がなければ「なし」）

## 環境変更

- （新しい環境変数、DBマイグレーション、設定ファイルの追加等）

```

## retrospective.md

```markdown
# Build {NN}: {タスク名} 振り返り

## セッション中にスキル外で受けた指示

（スキルの手順に含まれていないが、ユーザーから受けた指示・修正・補足を列挙する。
　スキル外の指示がなかった場合は「なし」）

## 改善提案

### R-1: （改善タイトル）
- **状況**: （どのステップで、どんな状態のときに発生したか）
- **指示内容**: （ユーザーからどんな指示・修正・補足があったか）
- **提案**: （何をどう改善すべきか）
- **優先度**: Must / Should / Could
- **対象**: スキル修正 / CLAUDE.md追記 / 記録のみ
- **具体案**: （対象ファイルに追記・修正する内容の文面案）

### R-2: ...
```

**記載ガイドライン:**
- スキルの手順通りに進んだ部分は書かない。**差分だけ**を記録する
- 「スキル外で受けた指示」が最重要のインプット。ユーザーが追加で指示したことは、スキルに欠けている情報の証拠
- 具体案は、そのまま適用できるレベルの具体性で書く
- 改善提案がなければ「改善提案なし」と記載する
