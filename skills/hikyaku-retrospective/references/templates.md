# 振り返りテンプレート集

## retrospective.md（新規作成）

フェーズに応じてヘッダーを変える:
- 企画フェーズ: `# 企画フェーズ 振り返り`
- 設計フェーズ: `# 設計フェーズ 振り返り`
- 実装フェーズ: `# Build {NN}: {ビルド名} 振り返り`

```markdown
# {フェーズヘッダー}

## セッション中にスキル外で受けた指示

（スキルの手順に含まれていないが、ユーザーから受けた指示・修正・補足を列挙する。
　スキル外の指示がなかった場合は「なし」）

## 改善提案

### R-1: （改善タイトル）
- **状況**: （どのステップで、どんな状態のときに発生したか）
- **指示内容**: （ユーザーからどんな指示・修正・補足があったか）
- **提案**: （何をどう改善すべきか）
- **優先度**: Must / Should / Could
- **対象**: （下記の分類から選択）
- **具体案**: （対象ファイルに追記・修正する内容の文面案）

### R-2: ...
```

### 「対象」フィールドの書式

| 分類 | 書式 | 例 |
|------|------|-----|
| スキル修正 | `skill:{スキル名}` | `skill:hikyaku-builder`, `skill:hikyaku-retrospective` |
| リポジトリインストラクション | `repo:{ファイル名}` | `repo:CLAUDE.md`, `repo:AGENTS.md` |
| ワークフローインストラクション | `workflow:instruction.md` | `workflow:instruction.md` |
| 記録のみ | `記録のみ` | `記録のみ` |

## retrospective.md（PRレビュー対応の追記）

既存の retrospective.md の末尾に以下のセクションを追加する:

```markdown

## PRレビュー対応の振り返り

### セッション中にスキル外で受けた指示

（PRレビュー対応中にスキルの手順に含まれていない指示を受けた場合に記載。なければ「なし」）

### 改善提案

#### R-N: （改善タイトル）
- **状況**: （どのステップで、どんな状態のときに発生したか）
- **指示内容**: （ユーザーからどんな指示・修正・補足があったか）
- **提案**: （何をどう改善すべきか）
- **優先度**: Must / Should / Could
- **対象**: （分類から選択）
- **具体案**: （対象ファイルに追記・修正する内容の文面案）
```

R-N の番号は既存の改善提案の続きから採番する。
