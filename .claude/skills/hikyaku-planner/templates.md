# 企画フェーズ テンプレート集

## questions.md

```markdown
# 企画段階の質問

### Q1: （質問内容）
**回答**: （ユーザーの回答）

### Q2: （質問内容）
**回答**: （ユーザーの回答）
```

## user-stories.md

```markdown
# ユーザーストーリー

## US-1: （ストーリー名）

**ユーザー**: （誰が）
**目的**: （何のために）
**操作**: （何をする）
**優先度**: Must / Should / Could
**受け入れ基準**:
- [ ] （条件1）
- [ ] （条件2）

## US-2: （ストーリー名）

...
```

**記載ガイドライン:**
- 優先度は MoSCoW法（Must / Should / Could / Won't）で分類する
- Won't は user-stories.md には記載しない（questions.md のスコープ外で言及済み）
- 受け入れ基準はユーザー視点で記述する（実装の詳細は含めない）

## retrospective.md

```markdown
# 企画フェーズ 振り返り

## セッション中にスキル外で受けた指示

（スキルの手順に含まれていないが、ユーザーから受けた指示・修正・補足を列挙する。
　スキル外の指示がなかった場合は「なし」）

## 改善提案

### R-1: （改善タイトル）
- **状況**: （どのステップで、どんな状態のときに発生したか）
- **指示内容**: （ユーザーからどんな指示・修正・補足があったか）
- **提案**: （何をどう改善すべきか）
- **優先度**: Must / Should / Could
- **対象**: スキル修正 / CLAUDE.md追記 / 記録のみ
- **具体案**: （対象ファイルに追記・修正する内容の文面案）

### R-2: ...
```

**記載ガイドライン:**
- スキルの手順通りに進んだ部分は書かない。**差分だけ**を記録する
- 「スキル外で受けた指示」が最重要のインプット。ユーザーが追加で指示したことは、スキルに欠けている情報の証拠
- 具体案は、そのまま適用できるレベルの具体性で書く
- 改善提案がなければ「改善提案なし」と記載する
